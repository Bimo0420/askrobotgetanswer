# Команда 2 — Построить приложение строго по plan.md

## Overview
Прочитать `plan.md`, верифицировать выбранные компоненты и источники, затем последовательно создать структуру проекта, установить зависимости и сгенерировать/скопировать компоненты строго в соответствии с планом. Никаких дополнительных компонентов.

## Steps
1. **Прочитать `plan.md`**  
   - Если файла нет — остановиться с ошибкой и предложить выполнить Команду 1.

2. **Подготовить окружение**  
   - Если проект не инициализирован: `pnpm create next-app@latest -- --experimental-app`  
   - Установить Tailwind (postcss, autoprefixer).

3. **Установить shadcn/ui через MCP**  
   - Использовать `mcp shadcn` для установки и генерации именно тех компонентов, что перечислены в `plan.md`.  
   - Если MCP недоступен — fallback к CLI (`npx shadcn@latest init`, затем `add`).

4. **Добавить/скопировать блоки из Aceternity и Cult**  
   - Скопировать указанные в `plan.md` компоненты в `components/aceternity/` и `components/cult/`.  
   - Адаптировать под Tailwind + shadcn theme.

5. **Создать layout'ы и страницы по плану**  
   - `app/layout.tsx` (RootLayout),  
   - `app/(auth)/layout.tsx` (AuthLayout),  
   - `app/(app)/layout.tsx` (AppLayout),  
   - `app/login/page.tsx`, `app/dashboard/page.tsx`, `app/documents/[id]/page.tsx`.

6. **Mock data**  
   - Создать мок-данные для документов (`lib/mocks/documents.ts`).  
   - Простая мок-авторизация (`lib/auth/mockAuth.ts`).

7. **Тестирование и проверка**  
   - Smoke check (все страницы открываются, компоненты работают).  
   - Проверка accessibility (tab navigation, aria labels).

8. **Коммиты / PR**  
   - Разбить работу на шаги (init, add shadcn, layouts, pages, aceternity, cult).  
   - PR с checklist из `plan.md`.

## Checklist
- [ ] `plan.md` прочитан и реализован.  
- [ ] shadcn-компоненты добавлены через MCP/CLI.  
- [ ] Aceternity блоки скопированы и адаптированы.  
- [ ] Cult блоки скопированы и адаптированы.  
- [ ] Страницы и layout'ы совпадают с `plan.md`.  
- [ ] Создан PR с checklist.
