# Команда 1 — Подготовить базу и создать plan.md

## Overview
Исследовать UI-регистры с помощью встроенных средств (особенно `mcp shadcn`), выбрать существующие компоненты, описать архитектуру, layout'ы, страницы и список компонентов. В конце — создать `plan.md` с пошаговым планом реализации.

## Steps
1. **Запросить список компонентов через MCP**
   - Вызвать `mcp shadcn` и получить:
     - Полный список доступных компонентов shadcn/ui.
     - Методы их использования (подключение, props, best practices).
   - Для Aceternity UI и Cult UI: открыть их официальные регистры/документацию и собрать список доступных блоков.  
   - Сохранить эти данные для планирования.

2. **Составить карту страниц и layout'ов**
   - `/login`  
   - `/dashboard`  
   - `/documents/[id]`  
   - RootLayout, AuthLayout, DashboardLayout, DocumentLayout

3. **Назначить конкретные компоненты для каждой страницы**  
   (строго из полученных списков MCP/регистров, без выдуманных).

4. **Определить стек и зависимости**
   - Next.js (app router)  
   - TailwindCSS  
   - shadcn/ui и рекомендуемые зависимости (class-variance-authority, clsx, lucide-react и др.)

5. **Сформировать `plan.md`**
   - Разделы: Overview, Pages, Layouts, Components (таблица), File structure, Step-by-step tasks, Acceptance criteria, PR checklist.  
   - Сохранить файл в корне проекта.

6. **Deliverable**
   - Файл `plan.md` в корне проекта.

## Checklist
- [ ] Вызван `mcp shadcn`, получен список компонентов.  
- [ ] Внесены компоненты из Aceternity UI и Cult UI.  
- [ ] Описаны страницы и layout'ы.  
- [ ] Создан `plan.md`.
